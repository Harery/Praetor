# Conditional Continue Protocol

## Reply Formats Accepted at Phase 3 Gate

### Format 1 — Unconditional continue
```
continue
```
All INFERRED items remain INFERRED. Phase 4 proceeds.

### Format 2 — Structured answers
```
continue with:
  Q1 = Yes, invoices are generated by an external cron service we'll integrate later
  Q2 = unknown
  Q3 = email retention = account_lifetime + 30 days
  Q4 = Stripe integration is in a separate repo, treat DEP-001 as DEFERRED
```

Effect:
- Q1, Q3, Q4 → register entries updated from INFERRED to CONFIRMED
- Q2 → remains INFERRED
- Phase 4 proceeds with updated registers

### Format 3 — Corrections
```
correct:
  M_AUTH module: also includes src/middleware/jwt-verify.ts (missed)
  BR-005: threshold should be 3, not 5 (we're stricter)
then continue
```

Effect:
- Orchestrator instructs A01/A02 to update; emits diff confirmation; proceeds.

### Format 4 — Scope override
```
override:
  RUN_MODULES = [M_AUTH, M_BILLING]
  RUN_PRIORITIES = [P0]
then continue
```

Effect:
- Run configuration updated; Phase 4 scoped accordingly.

### Format 5 — Combined
```
continue with:
  Q1 = unknown
  Q2 = confirmed
override:
  RUN_PRIORITIES = [P0, P1]
```

All three actions applied.

### Format 6 — Halt
```
halt
```

Or any message that doesn't start with `continue` / `correct` / `override`.
Orchestrator stops and waits.
